Take-Home Assignment: React Dashboard Builder

Goal

Build a simple dashboard editor where users can add and arrange chart widgets. Focus on clarity, interaction quality, and code structure over feature depth.

⸻

Tech Requirements
* React
* Any chart library (e.g., Recharts, Chart.js)
* Any layout/drag library (e.g., React Grid Layout)
* Frontend only (no backend)

⸻

Core Features

Layout
* Main dashboard canvas
* Right-hand sidebar with available widgets

Widgets
* Exactly 2 chart types:
    * Bar chart
    * Line chart

Interactions
* Add a chart by dragging from sidebar → canvas
* Rearrange charts within the dashboard
* Remove a chart from the dashboard
* Show empty state: “Drag a chart here”

Widget Customization (required)
Each widget must support one simple customization:

* Example: editable title

⸻

Functional Expectations
* Multiple widgets can be added
* Each widget maintains its own state
* Drag-and-drop behavior is clear and stable
* UI remains usable on smaller screens (no need for perfection)

⸻

Mock Data
* Use simple hardcoded or local JSON data
* No need for complex data modeling

⸻

Deliverables
* GitHub repo
* README including:
    * Setup instructions
    * Tech choices
    * Tradeoffs
    * What you’d improve with more time

⸻

Time Expectation
Please spend 2–5 hours max.
We care more about decision-making and clarity than completeness.

⸻

Evaluation Criteria
* Product thinking: Is the UI intuitive?
* React structure: Clean components, clear state flow
* Interaction quality: Drag/drop feels predictable
* UI quality: Clean and usable
* Communication: Clear README and tradeoffs

⸻

Notes:
Do NOT spend time on:
* Authentication
* Backend APIs
* Complex data logic
* More than 2 chart types
* You can use AI
